# 智备小语 - 部署上线指南（给指导老师）

> **项目名称**：智备小语 — K-12 语文教师智能备课系统  
> **技术栈**：Python 3.9 + Flask + waitress（生产服务器）  
> **前端**：纯静态文件（HTML/CSS/JS）  
> **AI 服务**：DeepSeek API  
> **端口**：5000

---

## 一、部署包内容说明

本压缩包包含以下文件结构：

```
zhibeixiaoyu/

│   ├── run_prod.py             # 生产启动入口（waitress）

│   ├── db/                     # 数据库模块（自动初始化）

│   └── knowledge/              # 教材知识库数据
├── zhibei_frontend/            # 前端静态文件
│   ├── index.html
│   ├── css/
│   └── js/
└── 部署指南.md                  # 本文件
```

## 二、环境变量（必须配置）

部署时需要设置以下环境变量：

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `DEEPSEEK_API_KEY` | DeepSeek API 密钥（必须） | `sk-xxxxxxxxxxxxxxxx` |
| `DEFAULT_AI_PROVIDER` | 默认 AI 服务商 | `deepseek` |
| `DEFAULT_AI_MODEL` | 默认 AI 模型 | `deepseek-chat` |
| `FRONTEND_DIR` | 前端文件目录 | `/app/zhibei_frontend` |
| `PORT` | 服务监听端口（云平台自动设置） | `5000` |

> ⚠️ `DEEPSEEK_API_KEY` 是唯一必须提供的变量，其余都有默认值。

## 三、部署方式 A：Docker 部署（推荐）

### 1. 安装 Docker

在服务器上安装 Docker（如已安装可跳过）：

```bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com | sh
sudo systemctl start docker
sudo systemctl enable docker

# CentOS
sudo yum install -y docker
sudo systemctl start docker
sudo systemctl enable docker
```

### 2. 上传部署包

将 `zhibeixiaoyu.zip` 上传到服务器并解压：

```bash
unzip zhibeixiaoyu.zip
cd zhibeixiaoyu
```

### 3. 构建并启动

```bash
# 构建镜像（设置 API Key）
docker build --build-arg DEEPSEEK_API_KEY=你的API密钥 -t zhibeixiaoyu .

# 启动容器
docker run -d -p 5000:5000 --name zhibeixiaoyu \
  -e DEEPSEEK_API_KEY=你的API密钥 \
  -e DEFAULT_AI_PROVIDER=deepseek \
  -e DEFAULT_AI_MODEL=deepseek-chat \
  -e FRONTEND_DIR=/app/zhibei_frontend \
  --restart always \
  zhibeixiaoyu
```

### 4. 验证

```bash
# 查看容器状态
docker ps

# 查看日志
docker logs zhibeixiaoyu

# 测试访问
curl http://localhost:5000
```

### 5. 配置域名（可选）

使用 Nginx 反向代理：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 四、部署方式 B：直接运行（无需 Docker）

### 1. 安装 Python 3.9

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install python3.9 python3.9-venv python3-pip -y

# CentOS
sudo yum install python39 -y
```

### 2. 上传并解压

```bash
unzip zhibeixiaoyu.zip
cd zhibeixiaoyu
```

### 3. 创建虚拟环境并安装依赖

```bash
cd zhibei_backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 4. 设置环境变量

```bash
export DEEPSEEK_API_KEY=你的API密钥
export DEFAULT_AI_PROVIDER=deepseek
export DEFAULT_AI_MODEL=deepseek-chat
export FRONTEND_DIR=../zhibei_frontend
```

### 5. 启动服务

```bash
python run_prod.py
```

服务启动后会监听 `http://0.0.0.0:5000`。

### 6. 后台运行（生产环境）

使用 systemd 服务实现开机自启：

```bash
# 创建服务文件
sudo nano /etc/systemd/system/zhibeixiaoyu.service
```

写入以下内容（**修改路径和 API Key**）：

```ini
[Unit]
Description=智备小语
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/zhibeixiaoyu/zhibei_backend
Environment=DEEPSEEK_API_KEY=你的API密钥
Environment=DEFAULT_AI_PROVIDER=deepseek
Environment=DEFAULT_AI_MODEL=deepseek-chat
Environment=FRONTEND_DIR=/opt/zhibeixiaoyu/zhibei_frontend
ExecStart=/opt/zhibeixiaoyu/zhibei_backend/venv/bin/python run_prod.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启用并启动：

```bash
sudo systemctl daemon-reload
sudo systemctl start zhibeixiaoyu
sudo systemctl enable zhibeixiaoyu

# 查看状态
sudo systemctl status zhibeixiaoyu
```

## 五、部署方式 C：云平台部署

如果使用云平台（如阿里云、腾讯云等），推荐使用 Docker 方式部署。

### 阿里云 ECS + Docker

参考方式 A 的步骤，在 ECS 实例上执行 Docker 部署即可。

### Railway（如之前尝试过）

1. 在 Railway 上新建项目
2. 关联 GitHub 仓库（或上传代码）
3. **Start Command 留空**（使用 Dockerfile 中的 ENTRYPOINT）
4. 设置环境变量 `DEEPSEEK_API_KEY`
5. 部署

### Render

1. 新建 Web Service
2. 关联 GitHub 仓库
3. **Start Command**: `cd zhibei_backend && python run_prod.py`
4. **Build Command**: `pip install -r zhibei_backend/requirements.txt`
5. 添加环境变量 `DEEPSEEK_API_KEY`

## 七、常见问题

| 问题 | 解决方案 |
|------|----------|
| 启动报错 `ModuleNotFoundError` | 确认 `pip install -r requirements.txt` 已执行 |
| 网页打开白屏/404 | 检查 `FRONTEND_DIR` 环境变量是否正确 |
| AI 功能不工作 | 检查 `DEEPSEEK_API_KEY` 是否正确配置 |
| 端口被占用 | 修改 `PORT` 环境变量或停止占用端口的程序 |
| 数据库报错 | 删除 `zhibei.db` 文件，系统会自动重新创建 |
